# CDC mTLS E2E Tests

End-to-end tests for CDC cross-cluster replication over mTLS.

## Architecture

3 Milvus clusters (A, B, C) run locally with `tlsMode=2` (mutual TLS), sharing the same CA, server cert, and client cert (`SAN=localhost`).

CDC reads per-cluster client certs from `milvus.yaml` paramtable via `tls.clusters.<clusterID>.{caPemPath,clientPemPath,clientKeyPath}`. Since cluster IDs contain dashes (`by-dev1`, etc.), these are configured in `milvus.yaml` (env vars can't represent dashes in keys). The `start_clusters.sh` script injects the config via `MILVUSCONF` pointing to a temp config dir. The replicate config only carries `uri` + `token` — no TLS fields in `connection_param`.

```
Cluster A (by-dev1)  proxy=19530  metrics=19191  cdc-metrics=29091
Cluster B (by-dev2)  proxy=19531  metrics=19192  cdc-metrics=29092
Cluster C (by-dev3)  proxy=19532  metrics=19193  cdc-metrics=29093
```

## Prerequisites

- Compiled Milvus binary with CDC mTLS support (the Go changes in this branch)
- `MILVUS_DEV_PATH` pointing to the repo root (with `bin/milvus`)
- `MILVUS_VOLUME_DIRECTORY` for logs/data storage
- Running etcd, pulsar, minio (via `scripts/setenv.sh`)
- Python env with `pymilvus` and `loguru`

## Quick Start

```bash
export MILVUS_DEV_PATH=/path/to/milvus
export MILVUS_VOLUME_DIRECTORY=/data/tmp/milvus-volumes

# 1. Start 3 mTLS clusters (auto-generates certs if needed)
./start_clusters.sh

# 2. Wait for clusters to be ready
curl -s http://localhost:19191/healthz  # Should return "OK"

# 3. Run tests
python test_mtls_replication.py
python test_mtls_switchover.py
python test_mtls_failure.py

# 4. Stop clusters
./stop_clusters.sh
```

## Tests

### test_mtls_replication.py

Configures A -> B, A -> C replication, then:
1. Creates collection on A, verifies replication to B and C
2. Inserts 2000 rows (two batches), verifies consistency
3. Drops collection, verifies propagation

### test_mtls_switchover.py

Reconfigures topology to B -> A, B -> C (B becomes primary), then:
1. Creates collection on B, verifies replication to A and C
2. Inserts 2000 rows, verifies consistency
3. Drops collection, verifies propagation

### test_mtls_failure.py

Failure/negative tests for mTLS enforcement:
1. **Plain connection** — no TLS → rejected
2. **One-way TLS** — no client cert → rejected
3. **Wrong CA** — server can't be verified → rejected
4. **Wrong client cert** — not signed by trusted CA → rejected
5. **CDC without per-cluster TLS** — restart CDC with `MILVUSCONF` pointing to original configs (per-cluster TLS commented out) → replication fails

## Files

| File | Description |
|---|---|
| `gen_certs.sh` | Generates 1 CA + 1 server cert + 1 client cert |
| `start_clusters.sh` | Starts 3 clusters with shared mTLS certs |
| `stop_clusters.sh` | Stops all Milvus processes |
| `common.py` | Constants, TLS clients, replicate config helpers |
| `collection_helpers.py` | Collection CRUD and replication wait helpers |
| `test_mtls_failure.py` | mTLS failure/negative tests (bad certs, CDC without TLS) |

## Certificates

All clusters share one cert set (generated by `gen_certs.sh`):

```
certs/
  ca.pem / ca.key        — CA (signs everything)
  server.pem / server.key — Server cert (SAN=localhost,127.0.0.1)
  client.pem / client.key — Client cert (used by CDC and PyMilvus)
```

TLS env vars in `start_clusters.sh` (server-side):
- `COMMON_SECURITY_TLSMODE=2` — enable mTLS
- `TLS_CAPEMPATH` — CA cert (verify peers)
- `TLS_SERVERPEMPATH` / `TLS_SERVERKEYPATH` — server identity

Per-cluster CDC outbound TLS (in `milvus.yaml` via `MILVUSCONF`):
```yaml
tls:
  clusters:
    by-dev1:
      caPemPath: /path/to/ca.pem
      clientPemPath: /path/to/client.pem
      clientKeyPath: /path/to/client.key
    by-dev2: ...
    by-dev3: ...
```
